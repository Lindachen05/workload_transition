# Workload Transition Analysis Handoff

Project title:

```text
Disruption or Arousal? Understanding the Joint Effect of Workload Transition Magnitude and Frequency on Human Performance
```

This document explains what each useful script studies, how to run it, what files it needs, what outputs it creates, and how the results can be used in the paper.

## 1. Research Logic

The project studies how workload transition **magnitude** and **frequency** jointly affect human performance.

Experimental factors:

```text
Magnitude / intensity:
LI = low-intensity workload transition
HI = high-intensity workload transition

Frequency:
LF = low transition frequency
MF = medium transition frequency
HF = high transition frequency

Main transition conditions:
LI_LF, LI_MF, LI_HF, HI_LF, HI_MF, HI_HF

Baseline:
BASE
```

The current performance analyses answer three levels of questions:

```text
Level 1: Are transition conditions worse than BASE?
Level 2: During each transition event, how strong is the immediate disruption and short-term recovery?
Level 3: Does prior transition exposure reduce resilience to a later disturbance?
```

The strongest current findings are:

```text
1. All transition conditions are worse than BASE on primary trial-level deviation metrics.
2. Event-level analyses show strong intensity effects on immediate disruption and peak response.
3. Frequency affects recovery success and residual error after transition events.
4. High-intensity transitions significantly worsen response to the next disturbance.
5. Delta RMSE shows a significant intensity x frequency interaction in the next-disturbance analysis.
6. Under high intensity, frequency shows a trend-level U-shaped pattern: LF worse, MF better, HF rebounds.
```

The 45 s tail/recovery analyses are weaker and should be treated as supplementary or diagnostic rather than the main result.

## 4. Main Analysis Scripts

### 4.1 Event-Level Immediate and Recovery Response

Script:

```text
run_02b_rq1_from_events.py
```

Run:

```powershell
E:\programfiles\anaconda\python.exe run_02b_rq1_from_events.py
```

Input:

```text
event_metrics_regular.csv
```

Outputs:

```text
rq1_trial_aggregates_from_events_2_3.csv
run_02b_rq1_from_events_results2
```

What this analysis studies:

```text
This analysis asks what happens immediately after regular workload transitions.
Instead of averaging across the whole trial, it focuses on transition events and
aggregates event-level responses to participant x condition.
```

Why it matters:

```text
This is the clearest analysis for immediate disruption and short-term recovery.
It directly supports RQ1 and RQ2 because it tests whether intensity and frequency
shape the performance response around transition events.
```

Method:

```text
1. Load event_metrics_regular.csv.
2. Keep the six transition conditions:
   LI_LF, HI_LF, LI_MF, HI_MF, LI_HF, HI_HF.
3. Aggregate event metrics to participant x condition.
4. Run repeated-measures ANOVA with:
   within-subject factors = intensity x frequency = 2 x 3.
```

Important metrics:

```text
imm_mean:
Immediate post-transition deviation. Higher values mean stronger immediate disruption.

rec_mean:
Recovery-period deviation. Higher values mean worse performance during recovery.

peak_mean:
Mean peak deviation in the early post-transition response window. Higher values mean a stronger transition shock.

delta_mean:
Average post-pre change around transitions. Higher positive values mean stronger deterioration after transition.

trec_med:
Median recovery time. Higher values mean slower recovery.

recover_success_rate:
Proportion of events that successfully recovered. Higher values mean better recovery.

resid_5_10s_mean:
Residual error in the 5-10 s post-transition window. Higher values mean more lingering error.
```

Main current results:

```text
imm_mean:
intensity F = 51.22, p < 0.001

peak_mean:
intensity F = 53.24, p < 0.001
intensity x frequency p = 0.066, trend-level

trec_med:
intensity F = 14.34, p = 0.0020

recover_success_rate:
frequency F = 10.52, p = 0.0004
intensity F = 44.88, p < 0.001

delta_mean:
frequency F = 11.76, p = 0.0002
intensity F = 17.80, p = 0.0009

resid_5_10s_mean:
frequency F = 12.38, p = 0.0001
```

How to interpret:

```text
Transition intensity mainly affects the immediate disruption: high-intensity transitions produce stronger immediate deviation, larger peaks, and longer recovery time.

Transition frequency is more clearly related to recovery success and residual error. This suggests frequency shapes how well participants stabilize after repeated transitions.
```

Use in paper:

```text
Use as a main result for RQ1/RQ2.
```

Suggested wording:

```text
Event-level analyses showed that transition magnitude strongly shaped immediate disruption and peak response, while transition frequency was more strongly associated with recovery success and residual error.
```

### 4.3 Next-Disturbance Resilience

Main analysis script:

```text
run_07_next_disturbance_resilience_fixed.py
```

Visualization script:

```text
visualize_next_disturbance_resilience.py
```

Run:

```powershell
E:\programfiles\anaconda\python.exe run_07_next_disturbance_resilience_fixed.py

```

Inputs for visualization:

```text
next_disturbance_response_fixed.csv

```

Outputs:

```text
figures/next_disturbance_resilience_publication.png
figures/next_disturbance_resilience_publication.pdf
next_disturbance_resilience_publication_summary.csv
```

What this analysis studies:

```text
This analysis asks whether previous transition exposure changes the participant's response to a later disturbance.
It is a resilience analysis: if a prior transition condition weakens recovery capacity, the next disturbance should produce larger error.
```

Target event logic:

```text
LI_LF / HI_LF:
analyze the 3rd transition event

LI_MF / HI_MF:
analyze the 5th transition event

LI_HF / HI_HF:
analyze the 7th transition event
```

Analysis window:

```text
pre-window:  5 s before the target transition
post-window: 5 s after the target transition
```

Important metrics:

```text
pre_rmse:
RMSE before the target disturbance.

post_rmse:
RMSE after the target disturbance. Higher means worse response.

delta_rmse:
post_rmse - pre_rmse. Higher means larger disturbance-induced deterioration.

peak_deviation:
Maximum deviation after the target disturbance. Higher means stronger peak disruption.

post_auc:
Accumulated error after the target disturbance. Higher means greater post-disturbance error burden.
```

Main statistical results:

```text
post_rmse:
intensity p = 0.00037

delta_rmse:
intensity p = 0.00203
intensity x frequency p = 0.0349

peak_deviation:
intensity p = 0.000069

post_auc:
intensity p = 0.00041
```

Pairwise result within high intensity:

```text
HI_LF > HI_MF for:
post_rmse, delta_rmse, peak_deviation

This means the low-frequency high-intensity condition produced the strongest next-disturbance impairment.
```

How to interpret:

```text
High-intensity transitions reduce resilience to later disturbances. The significant intensity x frequency interaction for delta_rmse suggests that the frequency effect depends on transition magnitude.
```

Use in paper:

```text
Use as a main result for RQ1/RQ2. This is one of the strongest analysis lines.
```

Suggested wording:

```text
High-intensity transitions significantly increased response magnitude to the subsequent disturbance. Delta RMSE further showed a significant intensity-by-frequency interaction, indicating that frequency modulated the resilience cost of high-intensity transitions.
```

### 4.4 U-Shaped Frequency Pattern

Script:

```text
visualize_next_disturbance_u_shape.py
```

Run:

```powershell
E:\programfiles\anaconda\python.exe visualize_next_disturbance_u_shape.py
```

Input:

```text
next_disturbance_response_fixed.csv
```

Outputs:

```text
figures/next_disturbance_u_shape.png
figures/next_disturbance_u_shape.pdf
next_disturbance_u_shape_summary.csv
```

What this analysis studies:

```text
This analysis tests whether the frequency pattern is U-shaped, especially under high-intensity transitions.
```

U-shape contrast:

```text
U = (LF + HF) / 2 - MF
```

Interpretation of the contrast:

```text
U > 0:
MF is lower than the average of LF and HF, which indicates a U-shaped pattern.

U = 0:
No quadratic/U-shaped pattern.

U < 0:
Inverted U-shaped pattern.
```

Main current results:

```text
delta_rmse, HI:
U = 4.81, p = 0.074

peak_deviation, HI:
U = 6.02, p = 0.084
```

How to interpret:

```text
The U-shaped pattern is visible under high intensity: LF is worst, MF is best, and HF rebounds. However, the U-shape contrast is trend-level rather than conventionally significant.
```

Use in paper:

```text
Use as a secondary interpretation of the significant delta_rmse intensity x frequency interaction. Do not claim a significant U-shape.
```

Suggested wording:

```text
Under high-intensity transitions, next-disturbance responses showed a trend-level U-shaped frequency pattern, with the lowest response under medium frequency and higher responses under low and high frequency.
```

### 5.2 Tail Mechanism Segmentation

Script:

```text
run_06_tail_mechanism_analysis.py
```

Outputs:

```text
tail_phase_segmentation.csv
tail_phase_rmanova.csv
tail_phase_pairwise.csv
```

What this analysis studies:

```text
This script segments the final 45 s tail into three phases:
damage, recovery, and steady state.
```

Phase definitions:

```text
Damage phase:
Tail start until rolling RMSE is sustained below a participant-specific damage threshold.

Recovery phase:
After damage ends, until rolling standard deviation is sustained below a participant-specific recovery threshold.

Steady phase:
Remaining time after recovery ends.
```

Important note:

```text
The comments mention 2 s / 4 s sustained windows, but the actual code uses:
DAMAGE_SUSTAIN_MS = 5000
RECOVERY_SUSTAIN_MS = 5000
```

Current status:

```text
Conceptually useful, but the current statistical results are weak. Treat this as exploratory/supplementary.
```

Use in paper:

```text
Only include if discussing exploratory recovery mechanisms. It is not recommended as a main result.
```

## 7. Mapping to Research Questions

### RQ1

Question:

```text
Do different combinations of workload transition magnitude and frequency lead to distinct levels of human performance in continuous tasks?
```

Useful evidence:

```text
run_02b_rq1_from_events.py
visualize_transition_worse_than_base.py
run_07_next_disturbance_resilience_fixed.py
visualize_next_disturbance_resilience.py
```

Answer supported by current results:

```text
Yes. Transition conditions generally worsen performance relative to BASE. Event-level responses show strong intensity and frequency effects. Next-disturbance responses show robust intensity effects and a significant intensity x frequency interaction for delta RMSE.
```

### RQ2

Question:

```text
How do workload transition magnitude and frequency shape immediate performance and delayed post-transition recovery?
```

Useful evidence:

```text
run_02b_rq1_from_events.py
diagnose_tail_45s_analysis.py
run_07_next_disturbance_resilience_fixed.py
visualize_next_disturbance_resilience.py
```

Answer supported by current results:

```text
Immediate performance is strongly affected by transition intensity. Recovery success and residual error are strongly related to transition frequency. The full 45 s tail shows only weak residual trends, but the next-disturbance analysis shows that prior transition exposure affects later resilience.
```

### RQ3

Question:

```text
What attentional disruption, reallocation, and recovery mechanisms underlie these performance effects?
```

Current status:

```text
The current scripts mainly analyze performance. They do not directly answer RQ3 without eye-tracking metrics.
```

Recommended eye-tracking metrics:

```text
baseline-corrected pupil diameter
blink rate
mean fixation duration
target AOI dwell proportion
off-target dwell proportion
AOI switching rate
gaze entropy
gaze recovery time
```

Recommended windows:

```text
pre-transition:       -5 s to 0 s
immediate response:    0 s to 5 s
early recovery:        5 s to 15 s
late recovery:        15 s to 45 s
```

Suggested RQ3 logic:

```text
If pupil dilation increases but gaze distribution remains stable, the effect may be more arousal-driven.

If gaze entropy, off-target dwell, or AOI switching increases, the effect suggests attentional disruption or reallocation.

If gaze/pupil metrics recover slowly, they can explain delayed performance impairment or reduced resilience.
```

## 9. Statistical Notes

Common thresholds:

```text
p < 0.05:
significant

0.05 <= p < 0.10:
trend-level / marginal
```

Important writing cautions:

```text
Do not write "significant U-shape" for the current U-shape analysis.
Write "trend-level U-shaped frequency pattern under high-intensity transitions."

The strongest defensible claims are:
1. transition conditions are worse than BASE;
2. event-level intensity effects are strong;
3. recovery success/residual error are frequency-sensitive;
4. high-intensity transitions reduce next-disturbance resilience;
5. delta RMSE has a significant intensity x frequency interaction.
```
